# Implementation Plan: Lead Management Backend (Supabase)

This plan outlines the steps to transition the **Outreach Generator** from a transient, CSV-based tool to a persistent, data-driven **Lead Management Dashboard**.

## 1. Goal
Move from "In-Memory/Google Sheets" data management to a permanent **Supabase** backend. 
- Save every lead uploaded via CSV.
- Save every AI-generated outreach sequence.
- Provide a persistent dashboard for tracking progress.

---

## 2. Supabase Database Schema

### `leads` Table
Stores contact information for every potential client.
| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | `uuid` | Primary Key (auto-generated) |
| `created_at` | `timestamp` | When the lead was added |
| `company_name` | `text` | Name of the pool company |
| `contact_person`| `text` | Owner/Manager name |
| `email` | `text` | Lead email address |
| `website` | `text` | Lead website URL |
| `status` | `text` | `new`, `generated`, `contacted`, `replied`, `ignored` |
| `metadata` | `jsonb` | Any extra fields from the CSV (Industry, City, Notes) |

### `outreach_sequences` Table
Stores the actual 4-email sequences generated by AI.
| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | `uuid` | Primary Key |
| `lead_id` | `uuid` | Foreign Key (links to a specific lead) |
| `generated_at` | `timestamp` | When the sequence was created |
| `emails` | `jsonb` | Array of 4 email objects (Subject, Body, Step) |

## 4. Why this is "Premium"
- **Speed**: You don't have to re-upload CSVs. Just open the app and pick up where you left off.
- **Organization**: No more messy Google Sheets. The status of every outreach attempt is visually clear.
- **Scalability**: You can eventually add "Sender Profiles" and "Scheduled Send" dates.
