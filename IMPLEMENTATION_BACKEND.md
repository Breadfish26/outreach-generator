# Implementation Plan: Lead Management Backend (Supabase)

This plan outlines the steps to transition the **Outreach Generator** from a transient, CSV-based tool to a persistent, data-driven **Lead Management Dashboard**.

## 1. Goal
Move from "In-Memory/Google Sheets" data management to a permanent **Supabase** backend. 
- Save every lead uploaded via CSV.
- Save every AI-generated outreach sequence.
- Provide a persistent dashboard for tracking progress.

---

## 2. Supabase Database Schema

### `leads` Table
Stores contact information for every potential client.
| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | `uuid` | Primary Key (auto-generated) |
| `created_at` | `timestamp` | When the lead was added |
| `company_name` | `text` | Name of the pool company |
| `contact_person`| `text` | Owner/Manager name |
| `email` | `text` | Lead email address |
| `website` | `text` | Lead website URL |
| `status` | `text` | `new`, `generated`, `contacted`, `replied`, `ignored` |
| `metadata` | `jsonb` | Any extra fields from the CSV (Industry, City, Notes) |

### `outreach_sequences` Table
Stores the actual 4-email sequences generated by AI.
| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | `uuid` | Primary Key |
| `lead_id` | `uuid` | Foreign Key (links to a specific lead) |
| `generated_at` | `timestamp` | When the sequence was created |
| `emails` | `jsonb` | Array of 4 email objects (Subject, Body, Step) |

---

## 3. Step-by-Step Implementation

### Phase 1: Infrastructure & Connection
1.  **Supabase Setup**:
    - Create a new Supabase project.
    - Run the SQL migration to create `leads` and `outreach_sequences` tables.
    - Enable **Row Level Security (RLS)** or set up a simple API key for personal use.
2.  **Frontend Setup**:
    - Install `@supabase/supabase-js`.
    - Create a `src/lib/supabase.ts` client to handle the connection.
    - Add `VITE_SUPABASE_URL` and `VITE_SUPABASE_ANON_KEY` to `.env`.

### Phase 2: The "Ingest" Logic (CSV → DB)
1.  **Refactor CSV Upload**:
    - Instead of just setting the state in `App.tsx`, we will now `upsert` (insert if new, update if exists) lead data into the `leads` table.
2.  **UI Feedback**:
    - Show a "Syncing to Dashboard..." loader during CSV processing.

### Phase 3: The "Persistence" Logic (AI Generation → DB)
1.  **Save Sequences**:
    - Update `sequenceGenerator.ts` to take a `leadId`.
    - After AI successfully generates the 4 emails, save them to the `outreach_sequences` table.
2.  **Auto-Update Status**:
    - Automatically change lead status from `new` to `generated` once a sequence is saved.

### Phase 4: The Lead Dashboard UI
1.  **Leads Tab**:
    - Create a new tab component: `src/components/LeadsDashboard.tsx`.
    - Fetch leads from Supabase on mount.
    - Display a premium, searchable table of all leads.
2.  **Sequence Drawer/Modal**:
    - When clicking on a lead, fetch their generated sequence (if any) and display it in a slide-over panel.
    - Give indicators if a sequence is missing (meaning they haven't been "AI-processed" yet).

### Phase 5: Future Proofing (Cleanup)
1.  **Manual "Mark as Sent"**: Add a button to mark a lead as "Contacted" locally.
2.  **Lead Export**: Allow exporting the *entire* dashboard (leads + their emails) as a single JSON or zip for backup.

---

## 4. Why this is "Premium"
- **Speed**: You don't have to re-upload CSVs. Just open the app and pick up where you left off.
- **Organization**: No more messy Google Sheets. The status of every outreach attempt is visually clear.
- **Scalability**: You can eventually add "Sender Profiles" and "Scheduled Send" dates.
